## Brand mark

Two color-inverted variants of the same geometric-K mark. The amber lower arm ("foot mid-stride / forward motion") is identical in both — only the tile and upper-arm colors swap so the mark stays high-contrast against either surface.

### Files
- `src/assets/brand/marks/koudou-mark-light.png` — **navy tile, white upper K arm, amber lower arm.** Used on light surfaces (landing nav, footer, favicon). This is the canonical hand-authored asset uploaded by the user.
- `src/assets/brand/marks/koudou-mark-dark.png` — **white tile, navy upper K arm, amber lower arm.** Used on the navy sidebar. Generated by `/tmp/invert_mark.py`, which classifies each pixel by nearest reference color (navy / white / amber) and swaps navy ↔ white while leaving amber pixels untouched.
- `public/favicon.png` — copy of the light variant.
- Brand lockdown doc: `src/assets/brand/BRAND.md`. Legacy mark iterations live under `src/assets/_archive/marks-legacy/` — never import from there.

### Wiring
- `src/components/AppSidebar.tsx` imports the **dark** variant via `BrandMark` (used in both expanded `h-8 w-8` and collapsed `h-9 w-9` modes).
- `src/pages/Landing.tsx` imports the **light** variant for nav (`h-9 w-9`) and footer (`h-5 w-5`).

### Usage rules
- Always render via `<img src={...} alt="Koudou" />` — do **not** rebuild as inline SVG.
- Pick the variant by surface, never by personal preference: dark variant on dark backgrounds, light variant on light backgrounds. Mixing them collapses contrast and the mark disappears (the bug that prompted this memory).
- Do **not** wrap the mark in an additional tile — the painted tile is part of the asset.
- To regenerate the dark variant after the light variant changes, re-run `/tmp/invert_mark.py`.
